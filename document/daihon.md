## ゲーム
アンケートやゲームで、基本的にやることは一緒（選択するとか回答するとかだけ）
ゲームは基本的には設定項目を作りこんで、
実際の進行時には出題→回答　の繰り返しのテンポだけを制御するのが理想かな。
ゲームは基本的には点数つけて、最終結果として名前：得点を発表する。

### チューニングガム

チューニングガムというアナログゲームそのままのゲーム。
ゲームの内容としては、「一流」から「四流」までと「おでんの具は？」とか答えがないお題がランダムに選ばれる。

これはテーブル内専用のゲーム。
問題数を入れると自動で進行する。（テーブル内の回答が揃うと自動でテーブル内の全員の回答が出る）

通常ゲームと匿名ゲームの二種類があって
匿名ゲームの場合は回答が揃ったらシャッフルして回答だけ表示して
誰がどの回答か当てるのも遊べる。（入力なしでテーブルでやればいいかも。→匿名回答公開のオプション）

誰かと同じ回答ならポイント。
10問くらいやってポイント数高い人を発表。結果発表は全体のみでいいかな。

### いい線行きましょう

いい線いきましょうというゲームをそのままやる。
お題がある「富士山は3100m、では大阪で一番高い山金剛山は何ｍ？」
「子供にあげるお年玉は何円？」
みたいなので、みんなの答えの「真ん中」が正解
全て数字で回答して、全員が揃ったら回答を開いて真ん中にポイント。

得点は距離ベースで決まる。
真ん中取れた人は100点がベースで、惜しいところの人も減衰ボーナスをもらえる。

const scores = sorted.map((_, idx) => {
  const i = idx + 1;
  const m = (n + 1) / 2;
  const d = Math.abs(i - m);
  const dMax = m - 1;

  if (d === dMax) return -30;  // 両端ペナルティ
  return Math.round(100 * (1 - d / dMax));  // 中央100→端に向かって減衰
});
```

7問くらいやってポイント数高い人を発表。結果発表は全体のみでいいかな。

### みんなのイーブン

みんなのイーブンをそのまんまやる。
お題（花見より花火のほうが好き、お金より自由の方が価値がある）みたいなやはり答えづらい質問に対して、イエスかノーかイーブンで答える。
出した札のイエス：ノーが2倍差ならイエスかノーが勝ち、イエスもノーも勝たない場合はイーブンを出した人が勝ち。

10問くらいやってポイント数高い人を発表。結果発表は全体のみでいいかな。

### くるっくりん（エクストリームスみたいなの。ラッキーナンバー、QWINTO、それ系）

1-15の数字がランダム（数字はようは箱に入ってるのでランダムだけど出るものは決まってる）に出てきて、
それを出る度に空きマスにあてはめていく。
この時、左から昇順になっているルールがあって、同値はいけるが低い数字を右には置けない。
マスは3列あって、
1列目は5マス、2列目は8マス、3列目は12マス。（仮に赤青黄としよう）
出る数字については 
1,2,14,15が3枚
3,13が4枚
4,12が5枚
5,11が6枚
6,10が7枚
7,8,9が8枚のデッキ。
んで、数字がめくられるたびに赤、青、黄色の3つのうちどれかの列に置くんだけど
次の数字は必ず前の列とは違うところに置かないといけない。
置けない時＝パスを押したときは -1点。
4回パスしたら終わり。最後まで残った人か埋めきった人がいたら順位が出る。

### メタストリームス

これもストリームス系。
1-99の数字のデッキからランダムに出てくる。
18マスあって、同じく4回パスで脱落。
最後まで残るか埋めきったら勝ち。
