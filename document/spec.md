# takenawa — 宴会進行アプリ 仕様書

## 概要

**takenawa（たけなわ）** は、宴会・パーティーの進行をリアルタイムに管理するWebアプリケーション。
管理者が「台本」に沿ってイベントを進行し、参加者はスマホでゲーム・アンケートに参加する。

- **技術スタック**: Next.js 15 / React 19 / TypeScript / Firebase Realtime Database / Tailwind CSS / GSAP
- **対象ブラウザ**: 参加者はスマホ、管理者はPC（横幅12カラムグリッド）

## 1. ルーム

### 1.1 ルーム作成

トップページ（`/`）から誰でもルームを作成できる。

| 項目 | 説明 |
|---|---|
| イベント名 | 「2026年 新年会」等 |
| イベント日 | YYYY-MM-DD |
| テーブル数 | 1以上の整数 |
| あいことば | 管理画面アクセス用パスワード（平文） |

ルームIDは `大文字2文字 + 数字5桁`（例: `AB12345`）で自動生成される。

### 1.2 認証モデル

| 対象 | 認証方式 |
|---|---|
| 全クライアント | Firebase Anonymous Auth（`auth != null` が必須） |
| 管理者 | あいことば一致 or creatorUid一致で自動認証 |
| 参加者 | localStorageの`player_{roomId}`で復帰 |

管理者認証はsessionStorageにキャッシュされ、同一タブ内では再入力不要。

### 1.3 マイルーム

Anonymous AuthのUIDをルーム作成時に保存（`config.creatorUid`）。
トップページで同一ブラウザから作成したルーム一覧を表示し、パスワード不要で管理画面に入れる。

## 2. 台本（シナリオ）

イベントの流れは「台本」として定義される。台本はステップの配列で、管理者が「次へ」「前へ」で進行する。

### 2.1 ステップ一覧

| タイプ | ラベル | 概要 |
|---|---|---|
| `entry` | 受付 | 参加者エントリー。常にStep 1固定。 |
| `table_game` | テーブルゲーム | テーブル単位のゲーム |
| `whole_game` | 全体ゲーム | 全参加者参加のゲーム |
| `break` | 歓談 | フリータイム |
| `participants` | 参加者一覧 | 同卓メンバーの表示 |
| `survey` | アンケート集計 | 選択肢式アンケート → 結果表示 |
| `survey_open` | アンケート回答依頼 | フリーテキスト収集 |
| `reveal` | 回答開示 | 任意ステップの回答を可視化 |
| `result` | 結果発表 | スコア集計（将来拡張） |
| `end` | 閉会 | 閉会メッセージ |

### 2.2 ステップ共通設定

すべてのステップに設定可能な項目:

| 項目 | 説明 |
|---|---|
| `label` | ステップ名（「チューニングガム」等） |
| `durationMinutes` | 所要時間（分）。経過時間タイマーに使用 |
| `display.message` | 参加者に表示するメッセージ。`{name}` `{tableNumber}` で変数置換 |
| `display.showTablemates` | 同卓メンバーを表示するか |
| `display.showFields` | 表示するエントリー項目ID |

### 2.3 台本編集

2つの編集モードがある:

**台本編集モード**: 「台本を編集」ボタンで全ステップをまとめて編集。ステップの追加・削除・並べ替え・タイプ変更が可能。保存時に以下の自動処理が走る:
- `survey` の後に `reveal`（棒グラフ）を自動挿入
- `survey_open` の後に `reveal`（一覧）を自動挿入
- `table_game` / `whole_game` の後に `reveal`（スコアボード）を自動挿入

**個別編集モード**: ステップを展開して「このステップを編集」で即時保存。進行中に微調整したい時に使用。

### 2.4 割り込みステップ

進行中に「割り込み」ボタンで、現在のステップの直後にステップを挿入できる。突発的なイベント（急遽のビンゴ等）に対応。

### 2.5 タイムキープ

各ステップの `durationMinutes` を設定すると、進行画面に経過タイマーが表示される。

- 緑色: `0:00 / 15:00` — 時間内
- 赤色: `16:30 / 15:00 (+1:30)` — 超過

各ステップの開始時刻は `state.stepTimestamps` に記録される。

## 3. 参加者管理

### 3.1 エントリー

参加者は `/?room={ルームID}` にアクセスし、エントリーフォームに入力して参加する。

#### エントリー項目

管理者が自由に定義できる動的フィールド:

| 属性 | 説明 |
|---|---|
| `id` | 内部識別子（`name`, `field_xxx`） |
| `label` | 表示名（「名前」「学校名」等） |
| `type` | `text` / `number` / `select` |
| `required` | 必須かどうか |
| `options` | select型の選択肢 |
| `showInHeader` | テーブル情報に表示するか |

`name` フィールドは必須で削除不可。他のフィールドは管理者がいつでも追加・編集・削除できる。
イベント途中で項目を追加した場合、既存参加者には `MissingFieldsForm` で追加入力を促す。

#### 卓表示設定

`showInHeader` トグルは `PlayersPanel` に常時表示され、進行中いつでも変更可能。
ONにしたフィールドは参加者のテーブル情報展開時に同卓者の名前の横に表示される。

### 3.2 テーブル割り当て

参加者のテーブル番号には3つの状態がある:

| 値 | 意味 |
|---|---|
| `0` | 未割当（新規参加者のデフォルト） |
| `1〜N` | テーブル番号 |
| `-1` | テーブル外（プール。ゲームに含めない） |

管理者は `PlayersPanel` でドラッグ操作的にテーブルを割り当てる。
「テーブルシャッフル」ボタンで割り当て済み参加者をランダム再配分。

### 3.3 テーブル情報プッシュ

管理者が「テーブル情報をプッシュ」を押すと、現在の割り当てがスナップショットとして `publishedTables.assignments` に保存される。

**重要**: プッシュされるまで参加者にはテーブル番号は見えない。管理者の作業中の割り当て変更は参加者に影響しない。

プッシュされると参加者のヘッダーにテーブルバッジと同卓者名が表示される。
参加者には自分のテーブルの情報のみが表示される（全テーブル一覧は見えない）。

プッシュ履歴は `publishHistory` に保存され、管理者のプレビュー画面で確認できる。

### 3.4 プレゼンス

Firebase `onDisconnect` フックで接続状態をリアルタイム追跡。
管理者画面で各参加者の横に接続状態インジケーター（緑/灰色のドット）を表示。

## 4. ゲーム

### 4.1 ゲーム分類

| カテゴリ | ゲーム | 入力方式 |
|---|---|---|
| Q&A式 | チューニングガム / いい線行きましょう / みんなのイーブン | テキスト・数値・選択 |
| カードめくり式 | くるっくりん / メタストリームス | ボード配置・パス |

### 4.2 Q&A式ゲームの進行

```
管理者: ゲーム開始
  ↓
管理者: お題を送信（プリセットまたは手入力）
  ↓
参加者: 回答を入力・送信
  ↓
管理者: 回答締切
  ↓
管理者: 回答を公開（全体 / テーブル / 個別）
  ↓
（繰り返し）
  ↓
管理者: スコアボード表示ON → 参加者に累積スコア表示
  ↓
管理者: 次のステップへ → ゲーム結果が自動保存
```

#### お題の状態遷移

```
open（回答受付中）→ closed（締切）→ revealed（結果発表）
```

#### 回答公開スコープ

| スコープ | 説明 |
|---|---|
| `all` | 全員の回答が全員に見える |
| `table` | 同テーブルの回答のみ見える |
| `players` | 指定したプレイヤーの回答のみ見える |

#### 匿名モード

ゲーム単位で `anonymousMode` を設定可能。ONにすると回答者名が `???` に置換され、回答順もシャッフルされる。

### 4.3 チューニングガム（`tuning_gum`）

同じ答えを出した人同士にポイントが入るゲーム。

**お題例**: 「三流のカレー」「おでんの具は？」
**入力**: テキスト（ひらがな推奨）
**スコアリング**:
- 回答を正規化（カタカナ→ひらがな、全角→半角、小文字化、トリム）
- 同じ回答の人数を数え、`(同回答人数 - 1)` がその人のスコア
- 例: 3人が「りんご」→ 各自 +2pt

### 4.4 いい線行きましょう（`good_line`）

全員の回答の中央値を狙うゲーム。

**お題例**: 「金剛山の標高は？」「お年玉はいくら？」
**入力**: 数値
**スコアリング**:
- 全回答を昇順ソートし、中央位置を基準に距離を計算
- 中央 = 100pt、端に向かって線形減衰、両端 = -30pt
- 式: `score = round(100 * (1 - distance / maxDistance))`
- 1人の場合は自動的に100pt

### 4.5 みんなのイーブン（`evens`）

Yes/No/Evenの3択で均衡を予測するゲーム。

**お題例**: 「花見より花火の方が好き？」
**入力**: 選択肢（Yes / No / Even）
**スコアリング**:
- Yes:Noの比率が2倍以上 → 多数派が +1pt
- Yes:Noの比率が2倍未満（均衡） → Even選択者が +3pt
- Even以外の回答のみで比率を計算

### 4.6 くるっくりん（`krukkurin`）

カードをボード上に昇順で配置していくゲーム。

**デッキ構成（80枚）**:
| 数字 | 枚数 |
|---|---|
| 1, 2, 14, 15 | 各3枚 |
| 3, 13 | 各4枚 |
| 4, 12 | 各5枚 |
| 5, 11 | 各6枚 |
| 6, 10 | 各7枚 |
| 7, 8, 9 | 各8枚 |

**ボード（3列・計25マス）**:
```
赤（5マス）:  [_][_][_][_][_]
青（8マス）:  [_][_][_][_][_][_][_][_]
黄（12マス）: [_][_][_][_][_][_][_][_][_][_][_][_]
```

**ルール**:
1. 管理者がカードをめくる → 数字 + ランダム得点（5〜25pt）が公開
2. 参加者は3列のいずれかの空きマスに配置、またはパス
3. **昇順制約**: 左から右に向かって数字が昇順（同値OK）
4. **異列制約**: 前回と同じ列には置けない
5. パス: -1pt。4回パスで脱落
6. 全マス埋め or 脱落 → 個人終了

### 4.7 メタストリームス（`meta_streams`）

くるっくりんの1列版。

**デッキ**: 1〜99 各1枚（計99枚）
**ボード**: 1列18マス
**ルール**: くるっくりんと同じ（異列制約はなし）

### 4.8 Streamsゲーム共通の進行

```
管理者: ゲーム開始 → デッキ生成＆シャッフル、全員のボード初期化
  ↓
管理者: 「めくる」 → カード公開、全員のacted=false
  ↓
参加者: マスをタップして配置 or 「パス」
  ↓
（全員acted → 管理者が次をめくる）
  ↓
管理者: 終了 or 全員脱落
```

管理者画面にはプレイヤー状況テーブル（名前・スコア・パス回数・アクション済み）がリアルタイム表示される。

### 4.9 プリセットお題

`questionBank.ts` にゲームタイプごとのお題を100問以上ストック。
管理者は「プリセット1問追加」「プリセット5問追加」で台本にランダム追加できる。
チューニングガムは「X流の〇〇」形式を自動生成（流の重みつきランダム）。

## 5. アンケート

### 5.1 アンケート集計（`survey`）

選択肢式のアンケートを実施し、結果を可視化する。

**設定**:
- 質問文
- 選択肢（1行に1つ）
- 複数選択許可フラグ

**参加者画面**: 選択肢をタップして回答。複数選択可の場合はトグル式。
**管理者画面**: ステップ展開時にリアルタイムで回答数/総人数と選択肢ごとの棒グラフ集計を表示。
**結果**: 台本保存時に `reveal`（棒グラフ）ステップが自動追加される。

### 5.2 アンケート回答依頼（`survey_open`）

フリーテキストで意見を収集する。

**設定**: 質問文のみ
**参加者画面**: テキストエリアで自由入力
**管理者画面**: ステップ展開時にリアルタイムで回答数/総人数と各回答テキスト（名前付き）を一覧表示。
**結果**: 台本保存時に `reveal`（一覧）ステップが自動追加される（不要なら手動削除可）。

### 5.3 回答開示（`reveal`）

任意のステップの回答・結果を自由なタイミングで可視化するステップ。

**設定**:
| 項目 | 説明 |
|---|---|
| 参照ステップ | どのステップのデータを使うか |
| 表示形式 | 一覧 / 棒グラフ / 円グラフ / スコアボード |
| スコープ | 全体 / テーブル / 特定プレイヤー |

`survey` / `survey_open` / `table_game` / `whole_game` のデータを参照可能。

## 6. メッセージ

管理者は進行中にいつでも参加者にメッセージを送信できる。

### 6.1 送信対象

| 対象 | 説明 |
|---|---|
| 全員 | 全参加者に表示 |
| テーブル指定 | 指定テーブルの参加者のみ |
| 個人指定 | 指定プレイヤーのみ |

### 6.2 表示

メッセージは送信時の `currentStep` に紐づき、タイムライン上の正しい位置に挿入される。
管理者名（`config.adminName`）が送信者名として表示される。

## 7. 参加者画面

### 7.1 構成

```
┌───────────────────────────┐
│    イベント名 / 日付        │ ← stickyヘッダー
│  ┌─────────────────────┐  │
│  │ テーブル3  中村（太字）  │  │ ← タップで展開
│  │  + 3名               │  │
│  ├─────────────────────┤  │
│  │ 田中（太字） 営業部    │  │ ← 同卓者 + showInHeaderフィールド
│  │ 佐藤（太字） 開発部    │  │
│  │ 鈴木（太字） 人事部    │  │
│  └─────────────────────┘  │
├───────────────────────────┤
│  ● Step1: 受付             │ ← タイムラインカード
│  │ エントリー完了！         │
│  │ テーブル3               │
│  │                         │
│  ● Step2: チューニングガム  │
│  │ お題: 三流のカレー       │
│  │ [回答入力]               │
│  │                         │
│  │ 📩 幹事: 楽しんでね！   │ ← 管理者メッセージ
│  │                         │
│  ● Step3: 歓談タイム       │
│  │ 歓談タイムです           │
│  ...                       │
└───────────────────────────┘
```

### 7.2 テーブル情報カード

ヘッダー内に角丸カードで表示。テーブルバッジ・自分の名前（太字白）・同卓者数をまとめて囲み、タップで同卓者リストを展開。名前は太字白、エントリー項目は通常白。

### 7.3 タイムラインスナップショット

テーブル移動があるイベントに対応するため、各ステップの表示データは「スナップショット」として凍結される。

- **現在のステップ**: ライブデータ（publishedTablesから同卓者を算出）
- **過去のステップ**: スナップショット（そのステップ時点のテーブル・同卓者を保存）
- **保存先**: localStorage（`timeline_{roomId}_{playerId}`）
- テーブルが変わった場合は「テーブルが変わりました: T1 → T3」通知を表示

### 7.4 アニメーション

- タイムラインカード: `animate-panel-in`（フェードイン）
- ゲーム回答: GSAP stagger（順番にフェードイン）
- スコアボード: canvas-confetti（紙吹雪）
- 回答ステータスバッジ: パルスアニメーション

## 8. 管理画面

### 8.1 3つのビュー

| タブ | レイアウト | 用途 |
|---|---|---|
| 全パネル | 左3+中6+右3 | 準備・確認 |
| 進行集中 | 左8+右4 | ゲーム進行時 |
| 参加者イメージ | 左8+右4 | プレビュー確認 |

### 8.2 ルーム情報パネル（左）

- QRコード表示＆A4印刷（5×7 = 35枚）
- イベント名・日付・管理者名のインライン編集
- テーブル数変更
- エントリー項目管理

### 8.3 台本・進行パネル（中央）

- **stickyバー**: 現在のステップ名・経過タイマー・前へ/次へ/割り込みボタン
- **ステップ一覧**: 展開で詳細表示。現在ステップは青ハイライト、過去は半透明
- **ゲーム操作**: ゲームステップ展開時にGameControls / StreamsControlsが表示
- **メッセージ送信**: パネル下部に常設
- **ツール**: テーブルシャッフル

### 8.4 参加者モニターパネル（右）

- テーブルごとの参加者一覧（接続状態ドット付き）
- 未割当・テーブル外の参加者
- テーブル割り当て操作
- プッシュボタン・プッシュ状態表示
- 卓表示設定（showInHeaderトグル）

### 8.5 参加者イメージパネル

スマホ風フレームで参加者画面を完全再現するプレビュー。
ドロップダウンで参加者を切り替え、その人の視点でタイムラインを確認できる。
アンケート入力・ゲーム状態・回答開示・スコアボードなど全コンテンツが表示される。

## 9. データフロー

### 9.1 Firebase パス構造

```
rooms/{roomId}/
  config/              … ルーム設定
  state/               … 進行状態（currentStep, phase, stepTimestamps）
  scenario/steps/      … 台本
  players/{playerId}/  … 参加者データ
  currentGame/         … 現在進行中のゲーム状態
    questions/{qId}/   … お題
    answers/{qId}/{playerId}/  … 回答
    streams/           … Streamsゲーム状態（デッキ・現在カード）
    boards/{playerId}/ … Streamsボード状態
  gameResults/{stepIdx}/  … 保存済みゲーム結果
  messages/{msgId}/    … 管理者メッセージ
  stepResponses/{stepIdx}/{playerId}/  … アンケート回答依頼
  publishedTables/     … 公開済みテーブル割り当て
  publishHistory/      … プッシュ履歴
```

### 9.2 リアルタイム購読

| 購読者 | 対象パス | 用途 |
|---|---|---|
| 管理者 | `rooms/{roomId}` | ルーム全体 |
| 管理者 | `rooms/{roomId}/players` | 参加者一覧 |
| 参加者 | `rooms/{roomId}` | ルーム全体（ステップ進行検知） |
| 参加者 | `rooms/{roomId}/players/{pid}` | 自分のデータ |
| 参加者 | `rooms/{roomId}/players` | 全参加者（同卓者算出用） |

### 9.3 データ永続化

| データ | 保存先 | タイミング |
|---|---|---|
| ゲーム結果 | `gameResults/{stepIdx}` | ステップ遷移時に自動保存 |
| アンケート回答依頼 | `stepResponses/{stepIdx}/{pid}` | 参加者が回答時 |
| スナップショット | localStorage | ステップ遷移時 |

## 12. 典型的なイベントフロー

### 準備（当日前）
1. ルーム作成（イベント名・日付・あいことば）
2. テーブル数設定
3. エントリー項目設定（名前 + 所属等）
4. 台本編集（ゲーム・アンケート・歓談を配置、お題プリセット追加）
5. QRコード印刷（卓上テントに貼る）

### 当日
1. 参加者がQRスキャン → エントリー
2. 管理者がテーブル割り当て（or シャッフル）
3. 「テーブル情報をプッシュ」→ 参加者にテーブル通知
4. 「次へ」で台本を進行
5. ゲームステップ: 開始 → 出題 → 締切 → 公開 → （繰り返し） → スコアボード表示
6. アンケートステップ: 参加者が回答 → 次ステップで結果表示
7. 歓談ステップ: 管理者からメッセージ送信
8. 途中のテーブル移動があればシャッフル → 再プッシュ
9. 閉会ステップでイベント終了
