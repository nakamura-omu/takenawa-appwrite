# 宴会ゲームアプリ 仕様書

## 概要

他大学との合同研修後の懇親会（居酒屋）で使用する、幹事進行型の宴会ゲームWebアプリ。
参加者のスマホからQRコードでアクセスし、管理者（幹事）が台本に沿って進行する。

## 技術スタック

| 項目 | 技術 |
|------|------|
| フロントエンド | Next.js (App Router) |
| リアルタイム同期 | Firebase Realtime Database |
| ホスティング | Vercel（無料プラン） |
| 言語 | TypeScript |
| スタイリング | Tailwind CSS |

## 想定規模

- 参加者: 20〜40名
- テーブル: 5〜10卓（4人1組）
- 同時接続: 最大50（参加者 + 管理者 + 予備）

## 画面構成

### 1. 管理者画面 `/admin`

幹事が操作する進行管理画面。PCまたはタブレットでの操作を想定。

#### 1-1. ルーム作成・設定

- ルームID（短い英数字、自動生成 or 手動設定）の作成
- テーブル数の設定（5〜10卓）
- 参加者QRコード表示（ルームURLを含む）
- カスタムエントリーフィールド設定
  - 参加者に入力を求める項目を自由に定義（追加・削除・並べ替え）
  - フィールドタイプ: テキスト / 数値 / セレクト（選択肢をカンマ区切りで指定）
  - 必須/任意の設定
  - デフォルト: 「名前」（テキスト、必須）
  - 受付開始前のみ編集可能

#### 1-2. 台本（シナリオ）ビュー

管理者画面のメインUI。ステップを順に進行する。

```
台本構成:
┌─────────────────────────────────────────┐
│ Step 1: 受付（参加者のエントリー待ち）      │
│ Step 2: テーブルゲーム ラウンド1           │
│ Step 3: テーブルゲーム ラウンド2           │
│ Step 4: テーブルゲーム ラウンド3           │
│ Step 5: 歓談タイム                       │
│ Step 6: 全体ゲーム                       │
│ Step 7: 結果発表                         │
│ Step 8: 閉会                            │
└─────────────────────────────────────────┘
```

- 各ステップに「次へ」ボタン
- 現在のステップがハイライト表示
- 各ステップに対してゲームモードの選択・お題の入力が可能
- 台本のステップは管理者が自由に追加・削除・並び替え可能
- **ステップ割り込み**: ライブ進行中に「割り込み」ボタンで新ステップをcurrentStep+1に即座に挿入（台本編集を開く必要なし）
  - ステップタイプ・ラベル・メッセージを設定可能
  - 「挿入後すぐに進む」オプション付き
- **進行コントロールはスティッキー固定**: パネル上部に張り付き、スクロールしても常にアクセス可能
- **表示モード切替**: ヘッダーに「全パネル」/「進行集中」タブ
  - 全パネル: ルーム情報(3) + 台本(6) + 参加者(3) の3カラム
  - 進行集中: 台本(8) + 参加者(4) の2カラム（ルーム情報を非表示）

#### 1-3. ゲーム進行パネル

各ゲームステップ実行中に表示される操作パネル。

テーブルゲーム時:
- お題の入力・送出
- 回答受付の開始/締切
- テーブルごとの回答一覧表示
- 結果（ポイント）の表示指示

全体ゲーム時:
- カードめくり / 数字発表
- 参加者の配置状況モニター
- スコア自動計算・ランキング表示指示

#### 1-5. 管理者メッセージ送信

進行中に参加者へメッセージを直接送信する機能。

- テキストエリア + ターゲット選択（全員 / テーブルN / 個人）
- 送信済みメッセージの履歴表示（折りたたみ）
- メッセージは参加者のタイムラインに黄色のカードとして表示

#### 1-6. ステップ入力プロンプト

各ステップに参加者入力を設定し、回答を収集・開示する機能。

- ステップ編集（個別/台本）で「入力プロンプトを追加」チェックボックス
- プロンプトテキスト・入力タイプ（text/number/select）・選択肢を設定
- 回答閲覧パネル（StepResponsesPanel）: 回答一覧をリアルタイム表示
- 開示コントロール（6パターン）:
  - 名前付き（全員）/ 名前付き（同テーブル）
  - 匿名（全員）/ 匿名（同テーブル）
  - 管理者のみ / 開示解除

#### 1-4. 参加者モニター・テーブル割り当て

- 接続中の参加者一覧（名前・テーブル番号・接続状態）
- テーブルごとのグループ表示
- 「未割当」エリア: テーブル未割当の参加者をまとめて表示
- テーブル割り当て操作: 参加者カードをクリック → テーブル番号を選択して割り当て
- テーブル間の移動: 割り当て済みの参加者も別テーブルに移動可能
- リアルタイム更新（Firebase経由で参加者画面にも即反映）

### 2. 参加者エントリー画面 `/?room=XXX`

- ルームIDの入力（QRコードからアクセスすれば不要）
- 管理者が定義したカスタムエントリーフィールドに基づく動的フォーム
  - フィールドタイプ: テキスト / 数値 / セレクト（選択肢）
  - 必須/任意の設定
  - 例: 「名前」「学校名」「年齢」「役職」など自由に設定可能
- テーブル番号は管理者が後から割り当て（参加者は選択しない）
- 「参加する」ボタン
- エントリー完了後: **スクロール型タイムライン表示**
  - 管理者の進行（currentStep）に連動して新しいカードが下に追加される
  - 各ステップの `display` 設定に基づき、メッセージ・テーブルメイト・フィールド情報を表示
  - テーブルメイト等の動的情報は localStorage にスナップショット保存（テーブル移動後も前の情報が残る）
  - ページリロード時は localStorage から復元
  - 最新カードへの自動スクロール
  - **管理者メッセージの統合表示**: ステップカードとメッセージカード（黄色）を時系列で統合
    - メッセージはターゲット（全員/自分のテーブル/自分宛）でフィルタリング
    - `sentDuringStep` でステップNの後、N+1の前に挿入
  - **ステップ入力フォーム**: `step.input` が設定された現在ステップに入力欄を表示
    - 送信後は「送信済み」に切替、リロード時は既存回答をチェックして復元
  - **開示された回答の表示**: `stepReveals` を監視し、named/anonymous/same_table に応じて表示

### 3. 参加者ゲーム画面 `/play`

管理者の進行に完全同期する受動的な画面。

- 待機中: 「まもなく始まります」等のメッセージ表示
- テーブルゲーム中: お題表示 → 回答入力 → 結果表示
- 全体ゲーム中: ゲームUI（後述）
- 歓談タイム中: 「歓談タイム中です 🍺」等の表示

## ゲームシステム

### カテゴリA: テーブルゲーム（4人1組）

テーブル内で盛り上がるアイスブレイク・コミュニケーション系。
管理者がラウンドごとにゲームタイプとお題を選択する。

#### A-1. 価値観マッチ

- **ルール**: お題に対して各自が自由回答 → 同テーブル内で一致（または類似）したらポイント
- **フロー**:
  1. 管理者がお題を送出（例:「朝ごはんといえば？」）
  2. 参加者が回答を入力（制限時間あり: 管理者が設定、デフォルト30秒）
  3. 制限時間終了 or 管理者が締切 → テーブル内の回答を一斉公開
  4. ポイント判定（完全一致 = 自動、類似 = 管理者判定 or テーブル内セルフジャッジ）
- **表示**: テーブル内メンバーの回答がカードめくり風に公開される
- **お題例**: 管理者がその場で入力 or プリセットお題集から選択

#### A-2. せーの！（同時オープン）

- **ルール**: お題に対して全員が同時に回答をオープン → 被ったらポイント
- **フロー**:
  1. 管理者がお題を送出（例:「大学あるあるといえば？」）
  2. 参加者が回答を入力（制限時間あり）
  3. 管理者が「オープン！」→ テーブル内の回答を同時公開
  4. 被り判定（完全一致 = 自動加点）
- **表示**: 全員の回答が同時にパッと表示される演出
- **お題例**: 管理者がその場で入力 or プリセットお題集から選択

#### テーブルゲーム共通仕様

- テーブルごとの累計ポイント表示
- テーブル対抗ランキング（管理者画面・結果発表時に全体投影用）
- ラウンド数は管理者が台本で自由に設定（推奨: 3〜5ラウンド）
- 各ラウンドでゲームタイプ（A-1 / A-2）を切り替え可能

### カテゴリB: 全体ゲーム（全員参加）

ビンゴの代替。全員が個別にプレイし、スコアで競う。
管理者がゲームタイプを選択する。

#### B-1. ストリームス

- **ルール**: 管理者が1〜30の数字カードを1枚ずつめくる。参加者はめくられた数字を自分の20マスのシートに配置。最終的に昇順に並んだ連続マスの長さでスコア計算
- **フロー**:
  1. 参加者に20マスの空シートを表示
  2. 管理者が「次のカード」ボタン → ランダムに数字が1枚公開（1〜30、各2枚ずつの計60枚から引く）
  3. 参加者は公開された数字を空いているマスに配置（制限時間: 管理者設定、デフォルト15秒）
  4. 全20枚配置完了後、自動スコア計算
- **スコア計算**:
  - 左から右に見て、昇順（同値含む）が連続しているマスの数をカウント
  - 連続数に応じたポイント表（例: 2連続=1pt, 3連続=3pt, 4連続=5pt, 5連続=7pt, ...）
  - 合計スコアで順位決定
- **表示**:
  - 参加者画面: 現在の数字 + 自分のシート + 残り枠数
  - 管理者画面: めくった数字の履歴 + 参加者の配置状況サマリー

#### B-2. 拡張枠（将来追加用）

- 管理者が全体ゲームのタイプを選択するUIだけ先に用意しておく
- B-1のストリームスをデフォルトとし、将来的に他のゲーム（ビンゴ、数字当て等）を追加可能な設計にする
- ゲームタイプごとに参加者画面のコンポーネントを差し替える構造

## データ構造（Firebase Realtime Database）

```
rooms/
  {roomId}/
    config/
      tableCount: number          // テーブル数
      createdAt: timestamp
      entryFields/                // カスタムエントリーフィールド定義
        {index}/
          id: string              // フィールドID（"name", "school" 等）
          label: string           // 表示名（"名前", "学校名" 等）
          type: string            // "text" | "number" | "select"
          required: boolean       // 必須フラグ
          options?: string[]      // type="select" の場合の選択肢

    state/
      currentStep: number         // 現在の台本ステップ番号
      phase: string               // "waiting" | "playing" | "result" | "break"
    
    scenario/                     // 台本定義
      steps/
        {stepIndex}/
          type: string            // "entry" | "table_game" | "whole_game" | "break" | "result" | "end"
          label: string           // 表示名（例: "アイスブレイク ラウンド1"）
          gameType?: string       // "value_match" | "seno" | "streams" | ...
          config?: object         // ゲーム固有の設定
          display?/               // 参加者表示設定
            message?: string      // 参加者に表示するメッセージ
            showTablemates?: boolean // テーブルメイト名を表示するか
            showFields?: string[] // 表示するフィールドID
          input?/                 // 参加者入力プロンプト設定
            prompt: string        // 表示するプロンプト（例: "好きな食べ物は？"）
            inputType: string     // "text" | "number" | "select"
            options?: string[]    // inputType="select" の場合の選択肢
    
    players/
      {playerId}/
        name: string
        tableNumber: number        // 0 = 未割当、1〜N = テーブル番号
        connected: boolean
        joinedAt: timestamp
        fields/                    // カスタムフィールドの入力値
          {fieldId}: string|number // 例: name="田中", school="○○大学", age=22
    
    currentGame/
      type: string                // 現在実行中のゲームタイプ
      round: number               // 現在のラウンド（テーブルゲーム時）
      
      // テーブルゲーム用
      question/
        text: string              // お題テキスト
        timeLimit: number         // 制限時間（秒）
        status: string            // "open" | "closed" | "revealed"
      answers/
        {playerId}/
          text: string
          submittedAt: timestamp
      
      // ストリームス用
      streams/
        deck: number[]            // カードデッキ（シャッフル済）
        drawnCards: number[]       // めくられたカードの履歴
        currentCard: number       // 現在表示中のカード
        cardIndex: number         // 何枚目か
        placements/
          {playerId}/
            grid: (number|null)[] // 20マスの配置状態
    
    messages/                     // 管理者メッセージ
      {messageId}/
        id: string
        text: string
        target/
          type: string            // "all" | "table" | "player"
          tableNumber?: number    // type="table" の場合
          playerId?: string       // type="player" の場合
        sentAt: timestamp
        sentDuringStep: number    // 送信時のcurrentStep（タイムライン並び順用）

    stepResponses/                // ステップ入力の回答
      {stepIndex}/
        {playerId}/
          value: string|number
          submittedAt: timestamp
          playerName: string      // 非正規化（表示用）
          tableNumber: number     // 非正規化（テーブル絞り込み用）

    stepReveals/                  // 回答の開示設定
      {stepIndex}/
        mode: string              // "named" | "anonymous" | "admin_only"
        target: string            // "all" | "same_table"

    scores/
      tables/
        {tableNumber}/
          total: number
          rounds/
            {roundIndex}: number
      players/
        {playerId}/
          total: number
          gameScores/
            {gameId}: number
```

## 画面遷移・状態管理

### 参加者側の画面状態

参加者画面は `rooms/{roomId}/state` を `onValue` でリアルタイム監視し、管理者の操作に完全追従する。

```
[エントリー画面]
    ↓ 参加完了
[待機画面] ←──────────────────────┐
    ↓ phase = "playing"          │
[ゲーム画面]                      │
    ├─ テーブルゲーム             │
    │   ├─ お題表示              │
    │   ├─ 回答入力              │
    │   └─ 結果表示              │
    ├─ 全体ゲーム               │
    │   ├─ カード配置             │
    │   └─ スコア表示             │
    ↓ phase = "break"            │
[歓談画面] ──────────────────────┘
    ↓ phase = "result"
[最終結果画面]
```

### 管理者側の操作フロー

```
[ルーム作成] → [台本編集] → [受付開始（QR表示）]
    → [台本に沿って「次へ」で進行]
        → テーブルゲーム: お題入力 → 回答受付 → 締切 → 結果公開
        → 全体ゲーム: カードめくり → (繰り返し) → スコア計算
        → 歓談: 特に操作なし
    → [結果発表] → [閉会]
```

## UI/UXガイドライン

### 参加者画面

- **スマホファースト**: 縦画面100%、片手操作可能
- **文字サイズ**: お題・数字は大きく表示（居酒屋の暗い照明でも見える）
- **操作は最小限**: 基本はタップのみ。テキスト入力はテーブルゲームの回答時のみ
- **ダークモード推奨**: 居酒屋の雰囲気に合う暗めのテーマ
- **アニメーション**: 回答オープン時、カードめくり時に軽い演出（派手すぎない）
- **接続断対応**: 再接続時に現在の状態に自動復帰

### 管理者画面

- **PC/タブレット向け**: 横画面、情報量多め
- **台本が常に見える**: 左側に台本ステップ一覧、右側にゲーム操作パネル
- **参加者状況が把握できる**: 接続人数、回答済み人数をリアルタイム表示
- **誤操作防止**: 重要な操作（結果公開、次のステップへ）は確認ダイアログ
- **プレビュー機能**: 参加者に見えている画面のプレビュー

## プリセットお題集

テーブルゲーム用のお題を管理者がすぐ使えるようにプリセットで用意する。
管理者はプリセットから選択 or 自由入力が可能。

### 価値観マッチ / せーの！ 共通

カテゴリ別に分類:

- **食べ物系**: 「朝ごはんといえば？」「ラーメンの味は？」「好きなコンビニスイーツは？」
- **大学あるある**: 「大学でいちばん面倒な手続きは？」「会議で使いがちなフレーズは？」「DXといえば？」
- **生活系**: 「休日の過ごし方は？」「最近ハマっていることは？」「旅行に行くならどこ？」
- **二択系（せーの！向き）**: 「きのこ vs たけのこ」「犬派 vs 猫派」

※ 管理者画面にお題集の一覧を表示し、ワンタップで選択できるUIにする

## 非機能要件

### パフォーマンス

- Firebase Realtime DBの無料枠（Sparkプラン）内で運用
  - 同時接続100まで、1GBストレージ、10GB/月転送
  - 20〜40人の宴会1回なら余裕で収まる
- Vercel無料プラン: 100GB帯域/月

### セキュリティ

- Firebase Realtime DBのセキュリティルール: 一時利用のため簡易設定で可
- ルームIDを知っている人のみアクセス可能（URLの推測困難性で担保）
- 個人情報は名前のみ（メールアドレス等は不要）
- Firebase Authentication は使用しない（匿名参加）

### 運用

- 宴会終了後にルームデータを削除する機能（管理者画面から）
- デプロイは `vercel` コマンドまたは GitHub push による自動デプロイ
